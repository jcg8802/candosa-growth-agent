2 Server Setup (Hetzner VPS)

Provision an always-on Linux server. Takes about 20 minutes.

2.1 — Create the Server

  1. Go to console.hetzner.cloud → New Project → "candosa-growth"
  2. Create Server → Location: Falkenstein (EU, close to Spain) → Image: Ubuntu 24.04 → Type: CX22 (€4.5/mo)
  3. Add your SSH key → Create
  4. Note the server IP address

2.2 — Initial Server Config

bash — run on your local machine
# SSH into your new server
ssh root@YOUR_SERVER_IP

# Update system
apt update && apt upgrade -y

# Install Node.js 22 (required for OpenClaw)
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs

# Install essential tools
apt install -y git curl wget unzip python3 python3-pip build-essential

# Create non-root user for running the agent
adduser --disabled-password --gecos "" cleo
usermod -aG sudo cleo

# Copy SSH keys to cleo user
cp -r /root/.ssh /home/cleo/.ssh
chown -R cleo:cleo /home/cleo/.ssh

# Switch to cleo user for all remaining steps
su - cleo

2.3 — Install OpenClaw

bash — run as cleo user on VPS
# Install OpenClaw globally
npm install -g openclaw

# Verify installation
openclaw --version

# Create workspace
mkdir -p ~/.openclaw/workspace
mkdir -p ~/.openclaw/workspace/memory
mkdir -p ~/.openclaw/workspace/data
mkdir -p ~/.openclaw/workspace/skills

# Initialize config (interactive — will prompt for API key)
openclaw init

2.4 — Install Python Dependencies

bash
pip3 install google-ads google-auth google-auth-oauthlib \
  google-api-python-client stripe requests python-dateutil

2.5 — Install gog CLI

bash — Homebrew on Linux
# Install Homebrew (works on Linux) /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" echo 'eval "$(/home/linuxbrew/.linuxbrew/bin/brew shellenv)"' >> ~/.bashrc source ~/.bashrc # Install gog brew install steipete/tap/gogcli # Verify gog --version

2.6 — Set Up as System Service (auto-restart on reboot)

bash — create systemd service
sudo tee /etc/systemd/system/cleo.service << 'EOF'
[Unit]
Description=Candosa Growth Agent (Cleo)
After=network.target

[Service]
Type=simple
User=cleo
WorkingDirectory=/home/cleo/.openclaw/workspace
ExecStart=/usr/bin/openclaw gateway start --foreground
Restart=always
RestartSec=10
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable cleo
sudo systemctl start cleo
sudo systemctl status cleo

6 Slack Integration Setup

Create a Slack app and connect it to OpenClaw. Takes ~15 minutes.

6.1 — Create the Slack App

  1. Go to api.slack.com/apps → Create New App → From scratch
  2. Name: Cleo · Workspace: Candosa's workspace → Create App
  3. Under OAuth & Permissions → Bot Token Scopes → Add:
    app_mentions:read · channels:history · channels:read · chat:write · im:history · im:read · im:write · reactions:read · users:read
  4. Under Event Subscriptions → Enable → Request URL: http://YOUR_VPS_IP:3000/slack/events
    Subscribe to bot events: app_mention · message.channels · message.im · reaction_added
  5. Under Socket Mode → Enable Socket Mode (easier than webhooks, no public URL needed)
  6. Under App-Level Tokens → Generate Token → Scope: connections:write → Copy the xapp- token
  7. Install to Workspace → Copy the xoxb- Bot Token
  8. Create channel #growth-ai and #paid-traffic in Slack → Invite Cleo bot to both

✅ With Socket Mode enabled, OpenClaw doesn't need a public URL — it maintains a websocket connection to Slack. Perfect for a VPS behind NAT.

Mandatory: GCLID Capture on Every Landing Page

Every landing page must capture the Google Click ID from the URL and store it in a cookie. Without this, conversion attribution is broken and Cleo can't import offline conversions to Google Ads.

JavaScript — add before </body> on every landing page
// GCLID capture — required for Google Ads attribution
(function() {
  const params = new URLSearchParams(window.location.search);
  const gclid = params.get('gclid');
  if (gclid) {
    // 90-day cookie — covers long consideration cycles
    document.cookie = 'gclid=' + gclid + '; max-age=7776000; path=/; SameSite=Lax';
    localStorage.setItem('gclid', gclid);
    localStorage.setItem('gclid_ts', Date.now());
  }
  // Also capture UTM params for channel attribution
  ['utm_source', 'utm_medium', 'utm_campaign', 'utm_content'].forEach(p => {
    const v = params.get(p);
    if (v) localStorage.setItem(p, v);
  });
})();

Content Production Workflow (Cleo-assisted)

  1. Keyword brief — Cleo pulls monthly volume + competition data from Semrush API, identifies content gaps
  2. Content brief — Cleo generates full brief: target keyword, search intent, suggested H1/H2/H3 structure, word count, internal links to add, competitors to outperform
  3. Draft — Human writer (or Cleo for first draft) writes the post following the brief
  4. Optimization — Run draft through Surfer SEO, hit target content score before publishing
  5. Review — Head of Growth approves. Native Spanish review for accuracy + tone.
  6. Publish — Publish to CMS. Cleo verifies it appears in Google Search Console within 48h.
  7. Distribute — Cleo auto-drafts LinkedIn post + IG caption from the article, schedules in Buffer
  8. Track — Cleo monitors ranking weekly, alerts if article drops >5 positions or gets a featured snippet opportunity
